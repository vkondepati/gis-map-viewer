# Security Policy

If you discover a security vulnerability in this project, please use GitHub Security Advisories for private reporting when possible. If you cannot use GitHub Security Advisories, open a private issue or contact the maintainers.

Recommended steps for reporters:

1. Provide a clear, reproducible description of the issue.
2. Include minimal reproduction steps and proof-of-concept if available.
3. Describe the potential impact and affected versions.

Maintainters will:

- Acknowledge the report promptly.
- Triage and validate the issue.
- Coordinate a fix and release with appropriate versioning and advisories.

Do not publicly disclose details until a fix or mitigation is available.

For further assistance, use the repository's GitHub Security Advisories or contact the maintainers via the GitHub repository contact methods.
