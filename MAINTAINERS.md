# NexaMap Maintainers

This file lists the current maintainers for NexaMap and explains how maintainers are added.

## Current Maintainers

- Venkat Kondepati — GitHub: [vkondepati](https://github.com/vkondepati) — Project owner, initial maintainer

## How to become a maintainer

1. Contribute high-quality code, documentation, or community support over time.
2. Be nominated by an existing maintainer or core contributor.
3. Approval by a majority of the Core Maintainers.

Maintainers are expected to:

- Review and merge PRs in a timely manner
- Triage issues and respond to community questions
- Help with releases and changelogs
- Enforce repository standards and Code of Conduct

## Stepping down

If a maintainer needs to step down, they should notify the Core Maintainers and, if possible, transfer responsibilities before leaving.

## Contact

For maintainer nominations or questions, open an Issue and tag `@vkondepati`.
